Berlekamp–Massey 算法

Berlekamp–Massey 算法是一种用于求数列的最短递推式的算法。给定一个长为 nnn 的数列，如果它的最短递推式的阶数为 mmm，则 Berlekamp–Massey 算法能够在 O(nm)O(nm)O(nm) 时间内求出数列的每个前缀的最短递推式。最坏情况下 m=O(n)m = O(n)m=O(n)，因此算法的最坏复杂度为 O(n2)O(n^2)O(n2)。

定义​

定义一个数列 {a0…an−1}\{a_0 \dots a_{n - 1} \}{a0​…an−1​} 的递推式为满足下式的序列 {r0…rm}\{r_0\dots r_m\}{r0​…rm​}：

∑j=0mrjai−j=0,∀i≥m\sum_{j = 0} ^ m r_j a_{i - j} = 0, \forall i \ge m∑j=0m​rj​ai−j​=0,∀i≥m

其中 r0=1r_0 = 1r0​=1。mmm 称为该递推式的 阶数。

数列 {ai}\{a_i\}{ai​} 的最短递推式即为阶数最小的递推式。

做法​

与上面定义的稍有不同，这里定义一个新的递推系数 {f0…fm−1}\{f_0 \dots f_{m - 1}\}{f0​…fm−1​}，满足：

ai=∑j=0m−1fjai−j−1,∀i≥ma_i = \sum_{j = 0} ^ {m - 1} f_j a_{i - j - 1}, \forall i \ge mai​=∑j=0m−1​fj​ai−j−1​,∀i≥m

容易看出 fi=−ri+1f_i = -r_{i + 1}fi​=−ri+1​，并且阶数 mmm 与之前的定义是相同的。

我们可以增量地求递推式，按顺序考虑 {ai}\{a_i\}{ai​} 的每一位，并在递推结果出现错误时对递推系数 {fi}\{f_i\}{fi​} 进行调整。方便起见，以下将前 iii 位的最短递推式记为 Fi={fi,j}F_i = \{f_{i, j}\}Fi​={fi,j​}。

显然初始时有 F0={}F_0 = \{\}F0​={}。假设递推系数 Fi−1F_{i - 1}Fi−1​ 对数列 {ai}\{a_i\}{ai​} 的前 i−1i - 1i−1 项均成立，这时对第 iii 项就有两种情况：

  1. 递推系数对 aia_iai​ 也成立，这时不需要进行任何调整，直接令 Fi=Fi−1F_i = F_{i - 1}Fi​=Fi−1​ 即可。
  2. 递推系数对 aia_iai​ 不成立，这时需要对 Fi−1F_{i - 1}Fi−1​ 进行调整，得到新的 FiF_iFi​。

设 Δi=ai−∑j=0mfi−1,jai−j−1\Delta_i = a_i - \sum_{j = 0} ^ m f_{i - 1, j} a_{i - j - 1}Δi​=ai​−∑j=0m​fi−1,j​ai−j−1​，即 aia_iai​ 与 Fi−1F_{i - 1}Fi−1​ 的递推结果的差值。

如果这是第一次对递推系数进行修改，则说明 aia_iai​ 是序列中的第一个非零项。这时直接令 FiF_iFi​ 为 iii 个 000 即可，显然这是一个合法的最短递推式。

否则设上一次对递推系数进行修改时，已考虑的 {ai}\{a_i\}{ai​} 的项数为 kkk。如果存在一个序列 G={g0…gm′−1}G = \{g_0 \dots g_{m' - 1}\}G={g0​…gm′−1​}，满足：

∑j=0m′−1gjai′−j−1=0,∀i′∈[m′,i)\sum_{j = 0} ^ {m' - 1} g_j a_{i' - j - 1} = 0, \forall i' \in [m', i)∑j=0m′−1​gj​ai′−j−1​=0,∀i′∈[m′,i)

并且 ∑j=0m′−1gjai−j−1=Δi\sum_{j = 0} ^ {m' - 1} g_j a_{i - j - 1} = \Delta_i∑j=0m′−1​gj​ai−j−1​=Δi​，那么不难发现将 FkF_kFk​ 与 GGG 按位分别相加之后即可得到一个合法的递推系数 FiF_iFi​。

考虑如何构造 GGG。一种可行的构造方案是令

G={0,0,…,0,ΔiΔk,−ΔiΔkFk−1}G = \{0, 0, \dots, 0, \frac{\Delta_i}{\Delta_k}, -\frac{\Delta_i}{\Delta_k}F_{k-1}\}G={0,0,…,0,Δk​Δi​​,−Δk​Δi​​Fk−1​}

其中前面一共有 i−k−1i - k - 1i−k−1 个 000，且最后的 −ΔiΔkFk−1-\frac{\Delta_i}{\Delta_k} F_{k-1}−Δk​Δi​​Fk−1​ 表示将 Fk−1F_{k-1}Fk−1​ 每项乘以 −ΔiΔk-\frac{\Delta_i}{\Delta_k}−Δk​Δi​​ 后接在序列后面。

不难验证此时 ∑j=0m′−1gjai−j−1=ΔkΔiΔk=Δi\sum_{j = 0} ^ {m' - 1} g_j a_{i - j - 1} = \Delta_k \frac{\Delta_i}{\Delta_k} = \Delta_i∑j=0m′−1​gj​ai−j−1​=Δk​Δk​Δi​​=Δi​，因此这样构造出的是一个合法的 GGG。将 FiF_iFi​ 赋值为 FkF_kFk​ 与 GGG 逐项相加后的结果即可。

如果要求的是符合最开始定义的递推式 {ri}\{r_i\}{ri​}，则将 {fj}\{f_j\}{fj​} 全部取相反数后在最开始插入 r0=1r_0 = 1r0​=1 即可。

从上述算法流程中可以看出，如果数列的最短递推式的阶数为 mmm，则算法的复杂度为 O(nm)O(nm)O(nm)。最坏情况下 m=O(n)m = O(n)m=O(n)，因此算法的最坏复杂度为 O(n2)O(n^2)O(n2)。

在实现算法时，由于每次调整递推系数时都只需要用到上次调整时的递推系数 FkF_kFk​，因此如果只需要求整个数列的最短递推式，可以只存储当前递推系数和上次调整时的递推系数，空间复杂度为 O(n)O(n)O(n)。

参考实现
vector<int> berlekamp_massey(const vector<int> &a) {
  vector<int> v, last;  // v is the answer, 0-based, p is the module
  int k = -1, delta = 0;

  for (int i = 0; i < (int)a.size(); i++) {
    int tmp = 0;
    for (int j = 0; j < (int)v.size(); j++)
      tmp = (tmp + (long long)a[i - j - 1] * v[j]) % p;

    if (a[i] == tmp) continue;

    if (k < 0) {
      k = i;
      delta = (a[i] - tmp + p) % p;
      v = vector<int>(i + 1);

      continue;
    }

    vector<int> u = v;
    int val = (long long)(a[i] - tmp + p) * power(delta, p - 2) % p;

    if (v.size() < last.size() + i - k) v.resize(last.size() + i - k);

    (v[i - k - 1] += val) %= p;

    for (int j = 0; j < (int)last.size(); j++) {
      v[i - k + j] = (v[i - k + j] - (long long)val * last[j]) % p;
      if (v[i - k + j] < 0) v[i - k + j] += p;
    }

    if ((int)u.size() - i < (int)last.size() - k) {
      last = u;
      k = i;
      delta = a[i] - tmp;
      if (delta < 0) delta += p;
    }
  }

  for (auto &x : v) x = (p - x) % p;
  v.insert(v.begin(), 1);

  return v;  // $\forall i, \sum_{j = 0} ^ m a_{i - j} v_j = 0$
}

朴素的 Berlekamp–Massey 算法求解的是有限项数列的最短递推式。如果待求递推式的序列有无限项，但已知最短递推式的阶数上界，则只需取出序列的前 2m2m2m 项即可求出整个序列的最短递推式。（证明略）

应用​

由于 Berlekamp–Massey 算法的数值稳定性比较差，在处理实数问题时一般很少使用。为了叙述方便，以下均假定在某个质数 ppp 的剩余系下进行运算。

求向量列或矩阵列的最短递推式​

如果要求向量列 vi\boldsymbol{v}_ivi​ 的最短递推式，设向量的维数为 nnn，我们可以随机一个 nnn 维行向量 uT\mathbf u^TuT，并计算标量序列 {uTvi}\{\boldsymbol{u}^T\boldsymbol{v}_i\}{uTvi​} 的最短递推式。由 Schwartz–Zippel 引理，二者的最短递推式有至少 1−np1 - \frac n p1−pn​ 的概率相同。

求矩阵列 {Ai}\{A_i\}{Ai​} 的最短递推式也是类似的，设矩阵的大小为 n×mn \times mn×m，则只需随机一个 1×n1 \times n1×n 的行向量 uT\mathbf u^TuT 和一个 m×1m \times 1m×1 的列向量 v\boldsymbol{v}v，并计算标量序列 {uTAiv}\{\boldsymbol{u}^T A_i \boldsymbol{v}\}{uTAi​v} 的最短递推式即可。由 Schwartz–Zippel 引理可以类似地得到二者相同的概率至少为 1−n+mp1 - \frac{n + m} p1−pn+m​。

优化矩阵快速幂​

设 fi\boldsymbol{f}_ifi​ 是一个 nnn 维列向量，并且转移满足 fi=Afi−1\boldsymbol{f}_i = A \boldsymbol{f}_{i - 1}fi​=Afi−1​，则可以发现 {fi}\{\boldsymbol{f}_i\}{fi​} 是一个不超过 nnn 阶的线性递推向量列。（证明略）

我们可以直接暴力求出 f0…f2n−1\boldsymbol{f}_0 \dots \boldsymbol{f}_{2n - 1}f0​…f2n−1​，然后用前面提到的做法求出 {fi}\{\boldsymbol{f}_i\}{fi​} 的最短递推式，再调用常系数齐次线性递推即可。

如果要求的向量是 fm\boldsymbol{f}_mfm​，则算法的复杂度是 O(n3+nlog⁡nlog⁡m)O(n^3 + n\log n \log m)O(n3+nlognlogm)。如果 AAA 是一个只有 kkk 个非零项的稀疏矩阵，则复杂度可以降为 O(nk+nlog⁡nlog⁡m)O(nk + n\log n \log m)O(nk+nlognlogm)。但由于算法至少需要 O(nk)O(nk)O(nk) 的时间预处理，因此在压力不大的情况下也可以使用 O(n2log⁡m)O(n^2 \log m)O(n2logm) 的线性递推算法，复杂度同样是可以接受的。

求矩阵的最小多项式​

方阵 AAA 的最小多项式是次数最小的并且满足 f(A)=0f(A) = 0f(A)=0 的多项式 fff。

实际上最小多项式就是 {Ai}\{A^i\}{Ai} 的最小递推式，所以直接调用 Berlekamp–Massey 算法就可以了。如果 AAA 是一个 nnn 阶方阵，则显然最小多项式的次数不超过 nnn。

瓶颈在于求出 AiA^iAi，因为如果直接每次做矩阵乘法的话复杂度会达到 O(n4)O(n^4)O(n4)。但考虑到求矩阵列的最短递推式时实际上求的是 {uTAiv}\{\boldsymbol{u}^T A^i \boldsymbol{v}\}{uTAiv} 的最短递推式，因此我们只要求出 AivA^i \boldsymbol{v}Aiv 就行了。

假设 AAA 有 kkk 个非零项，则复杂度为 O(kn+n2)O(kn + n^2)O(kn+n2)。

求稀疏矩阵行列式​

如果能求出方阵 AAA 的特征多项式，则常数项乘上 (−1)n(-1)^n(−1)n 就是行列式。但是最小多项式不一定就是特征多项式。

实际上如果把 AAA 乘上一个随机对角阵 BBB，则 ABABAB 的最小多项式有至少 1−2n2−np1 - \frac {2n^2 - n} p1−p2n2−n​ 的概率就是特征多项式。最后再除掉 det  B\text{det}\;BdetB 就行了。

设 AAA 为 nnn 阶方阵，且有 kkk 个非零项，则复杂度为 O(kn+n2)O(kn + n ^ 2)O(kn+n2)。

求稀疏矩阵的秩​

设 AAA 是一个 n×mn\times mn×m 的矩阵，首先随机一个 n×nn\times nn×n 的对角阵 PPP 和一个 m×mm\times mm×m 的对角阵 QQQ, 然后计算 QAPATQQ A P A^T QQAPATQ 的最小多项式即可。

实际上不用调用矩阵乘法，因为求最小多项式时要用 QAPATQQ A P A^T QQAPATQ 乘一个向量，所以我们依次把这几个矩阵乘到向量里就行了。答案就是最小多项式除掉所有 xxx 因子后剩下的次数。

设 AAA 有 kkk 个非零项，且 n≤mn \le mn≤m，则复杂度为 O(kn+n2)O(kn + n ^ 2)O(kn+n2)。

解稀疏方程组​

问题：已知 Ax=bA \mathbf x = \mathbf bAx=b, 其中 AAA 是一个 n×nn \times nn×n 的 满秩 稀疏矩阵，b\mathbf bb 和 x\mathbf xx 是 1×n1\times n1×n 的列向量。A,bA, \mathbf bA,b 已知，需要在低于 nωn^\omeganω 的复杂度内解出 xxx。

做法：显然 x=A−1b\mathbf x = A^{-1} \mathbf bx=A−1b。如果我们能求出 {Aib}\{A^i \mathbf b\}{Aib}(i≥0i \ge 0i≥0) 的最小递推式 {r0…rm−1}\{r_0 \dots r_{m - 1}\}{r0​…rm−1​}(m≤nm \le nm≤n), 那么就有结论

A−1b=−1rm−1∑i=0m−2Aibrm−2−iA^{-1} \mathbf b = -\frac 1 {r_{m - 1}} \sum_{i = 0} ^ {m - 2} A^i \mathbf b r_{m - 2 - i}A−1b=−rm−1​1​∑i=0m−2​Aibrm−2−i​

（证明略）

因为 AAA 是稀疏矩阵，直接按定义递推出 b…A2n−1b\mathbf b \dots A^{2n - 1} \mathbf bb…A2n−1b 即可。

同样地，设 AAA 中有 kkk 个非零项，则复杂度为 O(kn+n2)O(kn + n^2)O(kn+n2)。

参考实现
vector<int> solve_sparse_equations(const vector<tuple<int, int, int>> &A,
                                   const vector<int> &b) {
  int n = (int)b.size();  // 0-based

  vector<vector<int>> f({b});

  for (int i = 1; i < 2 * n; i++) {
    vector<int> v(n);
    auto &u = f.back();

    for (auto [x, y, z] : A)  // [x, y, value]
      v[x] = (v[x] + (long long)u[y] * z) % p;

    f.push_back(v);
  }

  vector<int> w(n);
  mt19937 gen;
  for (auto &x : w) x = uniform_int_distribution<int>(1, p - 1)(gen);

  vector<int> a(2 * n);
  for (int i = 0; i < 2 * n; i++)
    for (int j = 0; j < n; j++) a[i] = (a[i] + (long long)f[i][j] * w[j]) % p;

  auto c = berlekamp_massey(a);
  int m = (int)c.size();

  vector<int> ans(n);

  for (int i = 0; i < m - 1; i++)
    for (int j = 0; j < n; j++)
      ans[j] = (ans[j] + (long long)c[m - 2 - i] * f[i][j]) % p;

  int inv = power(p - c[m - 1], p - 2);

  for (int i = 0; i < n; i++) ans[i] = (long long)ans[i] * inv % p;

  return ans;
}
