欧拉图

本页面将简要介绍欧拉图的概念、实现和应用。

定义​

本文中仅讨论有限图。

在图论中，欧拉路径（Eulerian path）是经过图中每条边恰好一次的路径，欧拉回路（Eulerian circuit）是经过图中每条边恰好一次的回路。 如果一个图中存在欧拉回路，则这个图被称为欧拉图（Eulerian graph）；如果一个图中不存在欧拉回路但是存在欧拉路径，则这个图被称为半欧拉图（semi-Eulerian graph）。

限制 / 注意

此处定义中虽然使用「路径」一词，但严格说来此处使用的概念应该是「迹（trail）」。欧拉路径与欧拉回路仅能使用每条边恰好一次，但并没有对经过顶点的情况进行限制。

性质​

以下我们假设所讨论的图 GGG 中不存在孤立顶点。该假设不失一般性，因为对于存在孤立顶点的图 GGG，以下性质对从 GGG 中删除孤立顶点后得到的图 G′G'G′ 仍然成立。

对于连通图 GGG，以下三个性质是互相等价的：

  1. GGG 是欧拉图；
  2. GGG 中所有顶点的度数都是偶数（对于有向图，每个顶点的入度等于出度）；
  3. GGG 可被分解为若干条不共边回路的并。

以下我们对等价性进行证明。

若一个图 GGG 是欧拉图，那么 GGG 中所有顶点的度数都是偶数：考虑从任意顶点开始沿着欧拉回路走一圈，则每个点 vvv 的度数等于离开点 vvv 的次数加到达点 vvv 的次数。又由于行动的轨迹是一个回路，则对于每个点 vvv，离开该点的次数等于到达该点的次数。这也就是说，每个点的度数都形如 2k2k2k，即偶数。 特别地，对于有向图，根据相同的证明过程，每个顶点的入度等于出度。

若一个图 GGG 中所有顶点的度数都是偶数（或入度与出度相等），则它可被分解为若干条不共边回路的不交并：考虑从任意顶点 uuu 开始，选择任意出边 (u,v)(u, v)(u,v)，走向对应的相邻顶点 vvv 并删除 (u,v)(u, v)(u,v)，直到返回最初开始的顶点 uuu。可以证明该过程必定会最终回到 uuu：每当到达一个新的顶点 v≠uv \neq uv=u 时，根据上一条性质，该顶点剩余的度数为奇数，也就是说必定存在一条出边，该过程不会在点 vvv 终止。（换句话说，该过程会且仅会在回到点 uuu 时停止。）又因图 GGG 中的边数是有限的，该过程必定会在有限步内停止，则最终必然可以返回 uuu 并得到一条回路。注意到在前述证明中我们仅使用了点度数均为偶数的性质，且在找到并删除一条回路后剩下部分的图仍然满足该性质，我们可以不断重复该过程直到剩下的图为空图，从而将 GGG 拆分为若干条不共边的回路。 更进一步地，每条回路都可以被从其多次经过的顶点处分解成若干简单环的不交并，所以上述性质中的简单回路亦可被替换为简单环。

若一个连通图 GGG 可被分解为若干条不共边回路的不交并，则 GGG 是欧拉图：对于一组不共边回路，每次从中选出两条有共同顶点的回路并将其合并为一条，重复该过程直到不存在有共同顶点的两条回路。 可以证明该过程结束时剩下的回路唯一。对于任意两条不共边回路 P1,P2P_1, P_2P1​,P2​，若 P1P_1P1​ 与 P2P_2P2​ 共点，则可以在共点处直接进行合并；否则，任取 P1P_1P1​ 上的点 v1v_1v1​ 与 P2P_2P2​ 上的点 v2v_2v2​，根据 GGG 的连通性，存在连接 v1v_1v1​ 和 v2v_2v2​ 的路径 e1,e2,…,eke_1, e_2, \ldots, e_ke1​,e2​,…,ek​，其中的每条边 eie_iei​ 都被一个回路 CiC_iCi​ 包含，且 P1P_1P1​ 与 C1C_1C1​，CiC_iCi​ 与 Ci+1C_{i+1}Ci+1​，CkC_kCk​ 与 P2P_2P2​ 均存在共点（或者 Ci=Ci+1C_i = C_{i+1}Ci​=Ci+1​，此情况不影响证明）。此情况下，P1P_1P1​ 与 P2P_2P2​ 可以通过 C1,…,CkC_1, \ldots, C_kC1​,…,Ck​ 进行合并。也就是说，任意两条回路都可以进行合并，最后剩下的回路必定唯一，且组成该回路的边集是所有不共边回路的并即 E(G)E(G)E(G)，该回路为 GGG 上的欧拉回路，GGG 为欧拉图。

以上的性质同时也构成了欧拉图的判断条件。具体地说，一个图是欧拉图当且仅当非零度顶点互相（强）连通，且顶点的度数都是偶数（或入度与出度相等）。

对于半欧拉图，其性质与欧拉图相似：一个半欧拉图具有恰好两个奇度数的顶点，且这两个顶点就是欧拉路径的两个端点。通过将这两个点连接起来，可以将半欧拉图转化为欧拉图。通过删除欧拉图中的任意一条边，可以得到一个半欧拉图。 由此可以导出半欧拉图的判别法：一个图是半欧拉图当且仅当非零度顶点互相（强）连通，且奇度数顶点恰好有两个。对于有向图，第二个条件为恰存在两个顶点 u,vu, vu,v，其中 deg⁡+(u)−deg⁡−(u)=1,deg⁡+(v)−deg⁡−(v)=−1\deg^+(u) - \deg^-(u) = 1, \deg^+(v) - \deg^-(v) = -1deg+(u)−deg−(u)=1,deg+(v)−deg−(v)=−1，且其余顶点的入度等于出度。

欧拉回路/欧拉路径的构造​

此处我们介绍最常用的 Hierholzer 算法，该算法的核心思想为利用上述欧拉图性质中的第三点，即欧拉图可以被拆解为若干条不共边回路的并。 可以注意到，在上述证明中其实已经提到了完整可行的将不共边回路合并为欧拉回路的操作，且在使用合适的数据结构储存时（如使用类链表的结构储存环）实现并不困难。

算法的具体流程为先从图中找到一条回路作为当前回路，每次从当前回路中选取剩余度数不为零的点，从该点出发找到一条新的简单回路，并将该简单回路与当前回路合并，重复该过程直到当前回路中的所有点均无剩余度数，此时的当前回路即为欧拉回路。

该算法同样适用于有向图。对于半欧拉图，可以从图中找到一条连接两个奇度数点的路径作为当前路径，每次选取度数非零的点寻找简单回路并将其与当前路径合并，最后得到欧拉路径。

时间复杂度分析​

Hierholzer 算法的时间复杂度为 O(∣E∣+∣V∣)O(|E| + |V|)O(∣E∣+∣V∣)。

注意到在前述正确性分析中，在欧拉图或半欧拉图上寻找简单回路（或半欧拉图的初始路径）的过程是 无需回溯 的，只要沿着剩下的边一直走就必定可以发现所求的回路或路径，且 每条边仅会被访问一次。 为了利用这一性质，在实现上应采取类链表的方式储存图中的边，如邻接表或链式前向星，以便每条边在被访问过后即刻删除之。如果采用朴素的邻接矩阵进行储存，则每次寻边耗时 O(∣V∣)O(|V|)O(∣V∣)，总复杂度为 O(∣V∣∣E∣)O(|V||E|)O(∣V∣∣E∣)。

笔记

事实上，该算法的准确复杂度应为 O(∣E∣)O(|E|)O(∣E∣) 而非 O(∣V∣+∣E∣)O(|V| + |E|)O(∣V∣+∣E∣)，这是因为该算法的实现方式可以采取依赖于边而不依赖于点的方法，通过维护剩余边的总链表来进行下一步回路的寻找。

如果需要输出字典序最小的欧拉路或欧拉回路，则需要将边排序，时间复杂度为 Θ(∣E∣log⁡∣E∣)\Theta(|E|\log |E|)Θ(∣E∣log∣E∣) 或 Θ(∣E∣)\Theta(|E|)Θ(∣E∣)（使用计数排序或者基数排序）。

应用​

有向欧拉图可用于计算机译码。

设有 mmm 个字母，希望构造一个有 mnm^nmn 个扇形的圆盘，每个圆盘上放一个字母，使得圆盘上每连续 nnn 位对应长为 nnn 的符号串。转动一周（mnm^nmn 次）后得到由 mmm 个字母产生的长度为 nnn 的 mnm^nmn 个各不相同的符号串。

构造如下有向欧拉图：

设 S={a1,a2,⋯ ,am}S = \{a_1, a_2, \cdots, a_m\}S={a1​,a2​,⋯,am​}，构造 D=⟨V,E⟩D=\langle V, E\rangleD=⟨V,E⟩，如下：

V={ai1ai2⋯ain−1∣ai∈S,1≤i≤n−1}V = \{a_{i_1}a_{i_2}\cdots a_{i_{n-1}} |a_i \in S, 1 \leq i \leq n - 1 \}V={ai1​​ai2​​⋯ain−1​​∣ai​∈S,1≤i≤n−1}

E={aj1aj2⋯ajn−1∣aj∈S,1≤j≤n}E = \{a_{j_1}a_{j_2}\cdots a_{j_{n-1}}|a_j \in S, 1 \leq j \leq n\}E={aj1​​aj2​​⋯ajn−1​​∣aj​∈S,1≤j≤n}

规定 DDD 中顶点与边的关联关系如下：

顶点 ai1ai2⋯ain−1a_{i_1}a_{i_2}\cdots a_{i_{n-1}}ai1​​ai2​​⋯ain−1​​ 引出 mmm 条边：ai1ai2⋯ain−1ar,r=1,2,⋯ ,ma_{i_1}a_{i_2}\cdots a_{i_{n-1}}a_r, r=1, 2, \cdots, mai1​​ai2​​⋯ain−1​​ar​,r=1,2,⋯,m。

边 aj1aj2⋯ajn−1a_{j_1}a_{j_2}\cdots a_{j_{n-1}}aj1​​aj2​​⋯ajn−1​​ 引入顶点 aj2aj3⋯ajna_{j_2}a_{j_3}\cdots a_{j_{n}}aj2​​aj3​​⋯ajn​​。

这样的 DDD 是连通的，且每个顶点入度等于出度（均等于 mmm），所以 DDD 是有向欧拉图。

任求 DDD 中一条欧拉回路 CCC，取 CCC 中各边的最后一个字母，按各边在 CCC 中的顺序排成圆形放在圆盘上即可。

例题​

洛谷 P2731 骑马修栅栏

给定一张有 500 个顶点的无向图，求这张图的一条欧拉路或欧拉回路。如果有多组解，输出最小的那一组。

在本题中，欧拉路或欧拉回路不需要经过所有顶点。

边的数量 m 满足 1≤m≤10241\leq m \leq 10241≤m≤1024。

解题思路

本题为 Hierholzer 算法的直接应用。

保存答案可以使用 std::stack<int>，因为如果找的不是回路的话必须将那一部分放在最后。

注意，不能使用邻接矩阵存图，否则时间复杂度会退化为 Θ(nm)\Theta(nm)Θ(nm)。由于需要将边排序，建议使用前向星或者 std::vector 存图。示例代码使用 std::vector。
