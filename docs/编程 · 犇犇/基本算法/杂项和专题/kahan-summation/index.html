Kahan 求和

引入​

Kahan 求和 算法，又名补偿求和或进位求和算法，是一个用来 降低有限精度浮点数序列累加值误差 的算法。它主要通过保持一个单独变量用来累积误差（常用变量名为 ccc）来完成的。

该算法主要由 William Kahan 于 1960s 发现。因为 Ivo Babuška 也曾独立提出了一个类似的算法，Kahan 求和算法又名为 Kahan–Babuška 求和算法。

舍入误差​

在计算机程序中，我们需要用有限位数对实数做近似表示，如今的大多数计算机都使用 IEEE-754 规定的浮点数来作为这个近似表示。对于 13\frac{1}{3}31​，由于我们不能在有限位数内对它进行精准表示，因此在使用 IEEE-754 表示法时，必须四舍五入一部分数值（truncate）。这种 舍入误差（Rounding off error）是浮点计算的一个特征。

在浮点加法计算中，交换律（commutativity）成立，但结合律（associativity）不成立。也就是说，a+b=b+aa+b = b+aa+b=b+a 但 (a+b)+c≠a+(b+c)(a+b)+c \neq a+(b+c)(a+b)+c=a+(b+c)。因此在浮点序列加法计算中，我们可以从左到右一个个累加，也可以在原有顺序上，将他们两两分成一对。第二种算法会相对较慢并需要更多内存，也常被一些语言的特定求和函数使用，但相对结果更准确。

为了得到更准确的浮点累加结果，我们需要使用 Kahan 求和算法。

在计算 Snew=Sold+aS_{new}=S_{old}+aSnew​=Sold​+a（aaa 为浮点序列的一个数值）时，定义实际计算加入 SSS 的值为 aeff=Snew−Solda_{eff}=S_{new}-S_{old}aeff​=Snew​−Sold​, 如果 aeffa_{eff}aeff​ 比 aaa 大，则证明有向上舍入误差；如果 aeffa_{eff}aeff​ 比 aaa 小，则证明有向下舍入误差。则舍入误差定义为 Eroundoff=aeff−aE_{roundoff} = a_{eff} - aEroundoff​=aeff​−a。那么用来纠正这部分舍入误差的值就为 a−aeffa-a_{eff}a−aeff​, 即 EroundoffE_{roundoff}Eroundoff​ 的负值。定义 ccc 是对丢失的低位进行运算补偿的变量，就可以得到 cnew=cold+(a−aeff)c_{new} = c_{old} + (a - a_{eff})cnew​=cold​+(a−aeff​)。

Kahan 求和算法主要通过一个单独变量用来累积误差。如下方参考代码所示，sumsumsum 为最终返回的累加结果。ccc 是对丢失的低位进行运算补偿的变量（其被舍去的部分），也是 Kahan 求和算法中的必要变量。

因为 sumsumsum 大，yyy 小，所以 yyy 的低位数丢失。(t−sum)(t - sum)(t−sum) 抵消了 yyy 的高阶部分，减去 yyy 则会恢复负值（yyy 的低价部分）。因此代数值中 ccc 始终为零。在下一轮迭代中，丢失的低位部分会被更新添加到 yyy。

实现​

参考代码
float kahanSum(vector<float> nums) {
  float sum = 0.0f;
  float c = 0.0f;
  for (auto num : nums) {
    float y = num - c;
    float t = sum + y;
    c = (t - sum) - y;
    sum = t;
  }
  return sum;
}

编程语言的求和​

Python 的标准库指定了精确舍入求和的 fsum 函数可用于返回可迭代对象中值的准确浮点总和，它通过使用 Shewchuk 算法跟踪多个中间部分和来避免精度损失。

Julia 语言中，sum 函数的默认实现是成对求和，以获得高精度和良好的性能。同时外部库函数 sum_kbn 为需要更高精度的情况提供了 Neumaier 变体的实现，具体可见 KahanSummation.jl。
